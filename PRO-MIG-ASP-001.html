PRO-MIG-ASP-001 · Procedimiento Operativo · v2.0
Migración de Cliente
Local Manager Time → ASP
Procedimiento actualizado para la migración de una BD desde instalación local a la nube ASP de Manager. El proceso parte con una solicitud de evaluación de factibilidad, ejecutada íntegramente por Consultoría Técnica.
Consultoría Técnica Comercial / GEX IT Manager DTE (si aplica)

Comercial / GEX · Consultoría Técnica · IT Manager · DTE (condicional)
Estado
● VIGENTE
1
Objetivo y Alcance
Este procedimiento establece los pasos para la migración de una BD de cliente desde modalidad Local Manager Time hacia la modalidad ASP. Define las responsabilidades de cada área, los tiempos y los entregables, garantizando la integridad de los datos.
⚠
Este procedimiento aplica a la migración de una BD. Si el cliente migra más de una BD, cada BD adicional suma 2 sesiones más al proceso. Los plazos deben coordinarse directamente con Consultoría Técnica.
2
Solicitud de Evaluación de Factibilidad
El proceso se inicia cuando Comercial o GEX envía un correo a Consultoría Técnica con los datos del cliente y la información mínima para evaluar la factibilidad. Consultoría Técnica responde ese mismo correo indicando si la migración es factible y la cantidad de sesiones requeridas.
Correo de solicitud (Comercial / GEX → Consultoría Técnica)
Para: ayuda@manager.cl
Asunto: Cons_Técnico // [Nombre del Cliente]
Área: Comercial / GEX
Estimados, muy buenas tardes.

Junto con saludar, solicito evaluación de factibilidad de migración para el siguiente cliente:

Cliente: [Nombre del cliente]
RUT: [RUT]
Contacto: [Nombre y correo del contacto]

Datos para evaluación:
— Versión Manager Time instalada: [ej: v8.5.2]
— Versión SQL Server: [ej: SQL Server 2019]
— Cantidad de BDs a migrar: [N°]
— Formatos de impresión personalizados: [Sí / No — cuántos y de qué tipo]
— Integraciones con terceros: [Sí / No — detallar cuáles]
— ¿Tiene Acepta integrado con Manager?: [Sí / No]

Quedo atenta a sus comentarios. Saludos cordiales.
⚠
El asunto del correo debe seguir exactamente el formato Cons_Técnico // [nombre cliente] para que sea recepcionado correctamente por el área.
3
Respuesta de Factibilidad
El Consultor Técnico evalúa la información recibida y responde el correo indicando si la migración es factible y el número exacto de sesiones requeridas. Con esta respuesta, Comercial o GEX procede a crear la OT.
La respuesta de Consultoría Técnica incluye
✅
Factibilidad confirmada
Consultoría Técnica confirma si la migración es técnicamente factible con los datos entregados.
🔢
Número de sesiones
Cantidad exacta de sesiones requeridas. Base: 2 sesiones (Validación + Capacitación de Accesos) para 1 BD. Por cada BD adicional se suman 2 sesiones más.
📅
Plazos estimados
Consultoría Técnica indica los plazos estimados del proceso según la complejidad y cantidad de BDs.
⚠️
Consideraciones especiales
Cualquier condición especial detectada: integraciones complejas, formatos personalizados, requerimientos DTE.
ℹ
Con estos datos, Consultoría Técnica calcula la cantidad de sesiones requeridas. Por cada BD adicional se suman 2 sesiones más al total.
¿El cliente tiene Acepta con Manager?
⚠
Si el cliente factura electrónicamente con Acepta integrado a Manager, debe existir una sesión adicional con el área DTE. Esto debe quedar indicado en la OT.
4
Creación de OT
Con la respuesta de factibilidad de Consultoría Técnica, Comercial / GEX crea la OT indicando las sesiones calculadas.
La OT debe incluir
📋
DATO 1● Obligatorio
Cantidad de sesiones indicadas por Consultoría Técnica
Exactamente las sesiones que Cto. Técnica indicó en la respuesta de factibilidad. Base: 2 sesiones (Validación + Capacitación de Accesos) para 1 BD, + 2 sesiones adicionales por cada BD extra.
🔗
DATO 2● Si aplica
Sesión DTE si tiene Acepta con Manager
Si el cliente factura con Acepta integrado a Manager, agregar una sesión de DTE a la OT.
📝
DATO 3● Obligatorio
Destinatario: Consultoría Técnica
La OT se dirige a Consultoría Técnica.
5
Flujo de Migración
Con la OT aprobada, el Consultor Técnico ejecuta el proceso completo. Los pasos técnicos son los mismos que en la v1 del procedimiento.
⚙️
PASO 1● IT Manager
IT Manager crea el ambiente ASP
IT Manager crea la BD en el entorno ASP, el ambiente y los usuarios. Envía correo de bienvenida al cliente con credenciales de acceso.
BD creada en ASP · Correo bienvenida enviado al cliente
🔗
PASO 2● Cto. Técnico
Consultor Técnico entrega sitio de depósito al cliente
El Consultor Técnico entrega al cliente (o a su IT) el sitio donde debe depositar el backup. Se acuerda la fecha de corte — idealmente un jueves en la tarde.
Sitio entregado · Fecha de corte acordada
👤
PASO 3● IT del cliente
IT del cliente deposita el backup en el sitio
El IT del cliente genera el backup en la fecha de corte acordada y lo deposita en el sitio entregado por el Consultor Técnico. Esto ocurre el jueves PM.
Backup depositado en sitio
✅
PASO 4● Cto. Técnico
Consultor Técnico recepciona y valida el backup
El Consultor Técnico recepciona el backup, lo valida y lo traspasa a IT Manager para que lo suba a la BD ASP. Esto ocurre el viernes AM.
Backup validado y entregado a IT
⬆️
PASO 5● IT Manager
IT Manager sube el backup a la BD ASP
IT Manager recepciona el backup entregado por el Consultor Técnico, lo carga en la BD del entorno ASP y avisa cuando esté listo. Esto ocurre el viernes AM.
Backup cargado en ASP · Aviso a Cto. Técnico
🔍
PASO 6● Cto. Técnico
Consultor Técnico valida la BD y confirma sesión de validación
El Consultor Técnico verifica que la BD esté íntegra y correctamente cargada en el ambiente ASP. Una vez confirmado, avisa al cliente que el ambiente está listo y agenda la sesión de validación para el lunes AM.
BD validada y lista · Cliente notificado · Sesión agendada para lunes AM
📢
PASO 7● Cto. Técnico
Consultor Técnico conduce sesión de validación con el cliente — lunes AM
El Consultor Técnico conduce la sesión de validación con el cliente: revisión de BD, Ramaflex, formatos de impresión, importación/exportación y DTE si aplica. Se confirma que el cliente puede operar correctamente en el ambiente ASP.
Migración completada · Cliente operativo en ASP
✅
El cliente pierde únicamente el viernes para el ingreso de documentos al sistema — es el tiempo entre la generación del backup y la activación de la BD en ASP.
6
Sesiones de Ejecución con el Cliente
Consultoría Técnica conduce dos sesiones con el cliente para cerrar la migración: una de validación y otra de capacitación de accesos.
ℹ
El proceso de migración a ASP incluye 2 sesiones como parte de la ejecución: Validación y Capacitación de Accesos (Ramaflex). No incluye capacitación del sistema/uso ni sesiones adicionales. Si el cliente requiere capacitación de uso, debe coordinarse como un servicio independiente.
Sesión 1 · Validación
N° Ítem a validar Responsable
1
BD correctamente cargada en ASP
Verificar que los datos del cliente están íntegros y la BD está operativa.
 Cto. Técnico
2
Impresión
Verificar que los formatos de impresión funcionan correctamente en el nuevo ambiente ASP.
 Cto. Técnico
3
Proceso de Importación y Exportación
Validar los flujos de importación y exportación — es el cambio más relevante al pasar de Local a ASP. El cliente debe entender cómo funciona en el nuevo ambiente.
 Cto. Técnico
4
Sesión DTE (si aplica)
Si el cliente tiene Acepta con Manager, se realiza sesión adicional con el área DTE para validar la facturación electrónica en el nuevo ambiente.
 DTE (si aplica)
Sesión 2 · Capacitación de Accesos
N° Ítem a validar Responsable
1
Ingreso a Ramaflex
Mostrar al cliente cómo ingresar al entorno ASP a través de Ramaflex. Validar acceso de cada usuario.
 Cto. Técnico
⚠
Importación y exportación es el proceso que más cambia al migrar de Local a ASP. Es fundamental que el cliente lo entienda y valide en la Sesión 1.
