Uso exclusivo consultores
1
Previo a la sesión con el cliente
Configuración en Manager ERP

Responsabilidad exclusiva del consultor. Completar antes de llegar donde el cliente. Cada punto es requisito para el siguiente.

ℹ
Tipos de documento FAE y BOE
Se recomienda verificar que existan los tipos de documento FAE (Factura Electrónica) y BOE (Boleta Electrónica) en el ERP antes de la sesión, junto con los tipos de documento de cada forma de pago. No son bloqueantes para la parametrización, pero sí necesarios para emitir documentos electrónicos.
1
Crear grupos de usuarios
ERP › Usuarios › Grupos. Crear según roles: Caja, Ventas, Administrador. Los permisos de descuento del vendedor en GO dependen de este grupo.
2
Crear usuarios ERP
Asociar cada usuario a su grupo. El usuario GO lo crea el equipo de IT de Manager de forma independiente.
3
Verificar tipos de documento FAE y BOE
Verificar que existan FAE y BOE junto con los tipos de documento de cada forma de pago. Son necesarios para emitir documentos electrónicos, pero no impiden realizar la parametrización del POS.
4
Crear formas de pago de ventas
Módulo Ventas › Tablas › Formas de pago. Cada una debe tener tipo de documento contable asociado, incluyendo el crédito de clientes. Máximo 8 formas de pago.
5
Crear sucursales con datos completos
Manager › Tablas Generales › Sucursales. Completar todos los campos: RUT, N° Resolución, Fecha Resolución, Razón Social, Dirección, Comuna, Ciudad y Giro. Datos incompletos bloquean el funcionamiento.
6
Crear al menos un centro de costo
Requerido para poder crear terminales en GO. Módulo Finanzas › Centros de costo.
7
Crear vendedores en tabla de personal
Módulo Ventas › Personal. Asignar sucursal, bodega y lista de precios a cada vendedor. Las listas deben estar con IVA incluido.
8
Solicitar usuarios Manager GO a mesa de ayuda
Solicitar al equipo de IT la creación de los usuarios GO. Ellos crean el acceso de forma independiente y entregan las credenciales.
Para parametrizaciones usar la clave master disponible con soporte o solicitar un usuario administrador.
ℹ
RUT genérico para boletas
Crear en ERP un cliente con RUT 66.666.666-6. Debe estar creado antes de la implementación.
2
En el equipo del cliente
Instalación de Manager GO

Compatible con PC y Notebook con Windows. En Mac requiere Oracle VirtualBox con imagen Windows 7.

1
Ingresar por primera vez
Abrir Manager GO, ingresar email y contraseña. Si hay actualización disponible, el sistema la notifica automáticamente.
2
Ejecutar la actualización
Clic en Next. El progreso se muestra en porcentaje. Al finalizar se abre automáticamente el asistente de instalación.
3
Seguir el asistente de instalación
Clic en Siguiente. Puede solicitar clave de administrador del equipo (ej: Adm1n). Todas las actualizaciones requieren permisos de administrador.
4
Verificar versión instalada
Login › CONFIG (esquina superior izquierda). La barra muestra la versión, ej: Aplicaciones - v3.6.250. Para volver al login clic en "Manager Producción".
⚠
La actualización debe ejecutarse en cada equipo por separado
No es automática en todos los terminales. Todas requieren permisos de Administrador en el equipo.
Instalación en Mac (caso especial · VirtualBox)
A
Instalar Oracle VirtualBox 4.3.12
Descargar instalador para Mac. Ejecutar VirtualBox.pkg y seguir el asistente. Abrir VirtualBox desde la carpeta Aplicaciones.
B
Importar imagen Windows 7
Archivo › Importar servicio virtualizado. Indicar la ruta del archivo windows7.ova. Marcar "Reinicializar dirección MAC". Puede tardar 30+ minutos.
C
Ejecutar Manager GO desde la VM
La barra de tareas de Windows se oculta en el borde izquierdo. Acercar el mouse al borde izquierdo para mostrarla. Clic en el icono de Manager GO.
3
Hardware
Impresora Epson TM-T20

Impresora recomendada por Manager GO. Los modelos TM-T20iii y TM-T30 se conectan exclusivamente por red (RJ45), no por USB.

⚠
Cable de red RJ45 — el cliente debe proveerlo
Los nuevos modelos incluyen solo el cable de corriente. Confirmar disponibilidad antes de la visita.
Etapa 1 · Configuración de red
1
Conectar y encender
Cable RJ45 al router o switch. Instalar rollo de papel y verificar que avanza.
2
Detectar IP con Advanced IP Scanner
Descargar de advanced-ip-scanner.com/es. Explorar la red. Buscar "SEIKO EPSON". Anotar la IP.
3
Acceder a configuración web
Ingresar la IP en el navegador. Pasar el aviso de seguridad. Usuario: epson / Contraseña: epson.
4
Fijar IP estática y configurar Ethernet
Ethernet › 100Base-Tx Full Duplex › Send › Reset › Go Home. Luego TCP/IP › Manual › Send › Reset › Go Home.
Si la IP no se fija como estática, el router puede reasignarla y la impresión falla sin razón aparente.
Etapa 2 · Instalación de drivers
1
Descargar desde epson.cl
Soporte › Punto de venta › TM-T20II. Descargar "Driver de puerto virtual" (v8.6 o 8.7).
2
Instalar APDMANE_U
Ejecutar. Aceptar términos. Dejar "Manual English" y "Samples English". Install › Finish.
3
Instalar APD_455aE
Ejecutar. Aceptar. Seleccionar Minimum. Clic en Add. Seleccionar driver TM-T20 Receipt. Next › Finish.
4
Agregar impresora por TCP/IP en Windows
Panel de Control › Dispositivos e Impresoras › Agregar impresora › "No está en la lista" › Agregar por TCP/IP. Ingresar la IP. Si no la reconoce: Estándar › Generic Network Card.
Etapa 3 · Parametrización del layout
1
Abrir preferencias de impresión
Clic derecho sobre Epson TM-T20 › Preferencias de impresión › pestaña Layout (o Diseño).
2
Crear tamaño de papel personalizado
Paper Size › User Defined Paper Size. Nombre: Boleta. Ancho: 90.0 mm. Longitud: 310.0 mm. Save Paper Size › OK.
3
Configurar Layout
Paper Size: seleccionar "Boleta". Output Paper: 80mm Roll Paper. Horizontal Direction: -5.0 mm. Custom Scale: 91.50%.
4
Configurar Código 2D (timbre electrónico)
Pestaña Código 2D › Tipo: PDF417 › Configuración avanzada. Ancho módulo: 5 pts. Alto módulo: 5 pts. Nivel corrección: Nivel 0. Aceptar.
ℹ
Configurar también en Manager GO
Parámetros Generales › Parametrización de Boleta › activar Papel rollo. Tamaño papel Alto 279, Ancho 80. Líneas por documento: 20.
4
Junto al cliente
Configuración del Punto de Venta GO

Ingresar con usuario administrador. Ir al módulo Punto de Venta.

1
Crear terminales
Parámetros PV › Terminales. Un terminal por caja o punto de venta. Por cada uno: nombre, centro de costo, sucursal, cantidad de folios y mínimo de reabastecimiento.
2
Asignar terminal al equipo (Locales)
Pantalla principal › Locales › "Nombre terminal". Seleccionar el terminal del equipo actual. Esto vincula el terminal al PC por ID de dispositivo.
3
Asignar folios manuales
En Locales, ingresar rangos para documentos no electrónicos (Cotizaciones, NV): Desde, Actual y Hasta. Los campos Desde y Actual deben comenzar iguales.
4
Definir documentos a emitir
Parámetros Generales. Seleccionar documentos y si son electrónicos o pre-impresos. Factura y Boleta Exenta pueden marcarse como pre-impreso si no se usan.
5
Configurar formas de pago
Sección Medios de pago. Columna izquierda: nombre visible en el POS. Columna derecha: forma de pago del ERP. Máximo 8 + pago con nota de crédito.
6
Opciones adicionales
Envío en línea (obligatorio para facturadores electrónicos) · Ticket de cambio (solo BOV) · Puntos de fidelización · Documentos XL · Maneja lotes.
7
Parametrizar coordenadas de documentos
Parámetros Generales › Menú. Configurar posición de campos en cada tipo de documento impreso. Los parámetros se centralizan en el servidor.

5
DTE
Carga de folios electrónicos

Los folios son la autorización del SII para emitir documentos tributarios. Deben cargarse antes de operar.

1
Descargar XML desde sii.cl
El cliente descarga los archivos XML de folios. Un archivo por tipo de documento: boletas, facturas, notas de crédito.
2
Enviar a mesa de ayuda para conversión
Mesa de ayuda transforma el XML al formato AES compatible con Manager GO.
3
Cargar archivo AES en GO
Parámetros Generales › Cargar Archivo ACEPTA. Cargar un archivo a la vez. El archivo AES puede editarse con Notepad++ para seleccionar el rango antes de cargarlo.
4
Verificar distribución entre terminales
Los folios quedan en el PC donde se cargaron. Al conectarse, cada terminal toma automáticamente el rango configurado.
⛔
Restricciones críticas
No se puede cargar un archivo AES ya cargado ni uno con folios en un rango existente. Si los folios se agotan sin internet, el sistema no permite guardar documentos. Siempre mantener folios de repuesto.
6
Operación diaria
Uso del sistema

Flujo completo desde la apertura hasta el cierre de caja.

Apertura de caja
1
Primera sincronización
Al ingresar por primera vez en un equipo, el sistema sincroniza la base local: productos, clientes, tablas. Puede tardar varios minutos.
2
Apertura de caja
Menú › Apertura de caja. Ingresar usuario y monto inicial en efectivo. Aparece en el informe de cierre como "monto de apertura".
Emisión de documentos
1
Seleccionar tipo de documento y datos del cliente
Elegir: Boleta, Factura, Cotización, NV o Guía. Ingresar vendedor y cliente (ambos obligatorios). Para boletas sin identificar usar el RUT genérico.
2
Agregar productos
Buscar por código, código de barras o descripción. ENTER o clic en el carro. Se puede aplicar descuento por ítem o global. Para ver stock multisucursal, clic en el ícono de caja al buscar.
3
Registrar forma de pago
Clic en Forma de pago. Admite pago mixto. Para efectivo, ingresar el monto recibido y el sistema calcula el vuelto.
4
Guardar e imprimir
Clic en Guardar. El documento se emite e imprime. El sistema pregunta si reimprimir o continuar.
Cierre de caja
1
Ciclo documentario
Desde una Nota de Venta se puede generar Boleta o Factura usando el botón Ciclo de documentos.
2
Cierre de caja
Menú › Cierre de caja. Genera informes de arqueo con resumen de ventas por forma de pago.

Soporte técnico avanzado
Corrección de Folios y KNA

Procedimiento derivado de la sesión del 8 de mayo de 2026 (Gabriel Clavijo). Aplica cuando hay inconsistencias en la asignación de rangos de folios electrónicos.

⚠
Causa raíz más frecuente
Usar el mismo usuario Go en múltiples equipos simultáneamente puede generar colisiones en la asignación de folios. Cada terminal debe tener su propio usuario Go.
Las 3 tablas involucradas
TablaNombre en BDQué contieneCuándo revisarla
T1 · Pool total ACEPTA_DB Folios descargados del SII. Registra rango desde/hasta, tipo de documento y último usado. Si "último usado" = "hasta", el sistema considera el rango agotado. Siempre primero.
T2 · Terminales TERMINAL_DB Nombre, sucursal, centro de costo, estado e ID del dispositivo (identificador único del equipo Windows). Para obtener el ID interno de los terminales con problemas.
T3 · Rangos RANGOS_ELECT Rangos de folios asignados a cada terminal: desde/hasta, último usado y estado. Aquí ocurren la mayoría de los errores. Diagnóstico de secuencia y duplicados.
Procedimiento de diagnóstico
1
Revisar tabla ACEPTA — estado del pool
Verificar que "último asignado" no haya llegado al valor "hasta". Si son iguales, el sistema bloquea la asignación de nuevos folios.
SQL
SELECT * FROM acepta_db -- Verificar campos: desde, hasta, ultimo_asignado, estado -- Si ultimo_asignado = hasta → rango considerado agotado
2
Identificar terminales con problemas en tabla TERMINAL
Anotar el ID interno de cada terminal afectado.
SQL
SELECT * FROM terminal_db ORDER BY id_terminal
3
Revisar tabla RANGOS_ELECT — secuencia completa
Ordenar por rango_d. Verificar: rango corresponde al lote actual, "último usado" es coherente, rangos consecutivos sin huecos ni solapamientos.
SQL
SELECT * FROM rangos_elect ORDER BY rango_d

Corrección Terminal 19 — registro duplicado en KNA

La KNA tenía dos registros en estado "en uso" para el mismo terminal. Al reimprimir, el sistema no actualizaba el puntero y el folio quedaba bloqueado. Solución: eliminar el registro duplicado.

SQL
-- Eliminar registro duplicado (id_rango del registro incorrecto) DELETE FROM rangos_elect WHERE id_rango = 1552 -- Verificar que quedó solo un registro en uso SELECT * FROM rangos_elect WHERE id_terminal = 19 AND estado = 'en uso'
✓
Resultado esperado
El terminal toma el folio 85.051 correctamente y la secuencia continúa sin problemas.
Corrección Terminal 20 — valor incorrecto en "último usado"

El campo "último usado" apuntaba a 87.050 en lugar de 73.050 (el "hasta" del rango antiguo). Esto impedía que el sistema asignara el rango 87.051–89.050 porque lo consideraba ya ocupado. Solución: actualizar el campo al valor correcto.

SQL
-- Corregir último_usado al valor del rango antiguo UPDATE rangos_elect SET ultimo_usado = 73050 WHERE id_rango = 1537 -- Verificar resultado SELECT id_rango, id_terminal, rango_d, rango_h, ultimo_usado, estado FROM rangos_elect WHERE id_terminal = 20 ORDER BY rango_d
✓
Resultado esperado
El sistema reconoce el rango 72.051–73.050 como agotado y asigna automáticamente el siguiente: 87.051–89.050.
KNA local — cómo corregirla
ℹ
Servidor vs. KNA local
Error en todos los equipos para el mismo terminal → problema en el servidor. Error en un solo equipo → problema en la KNA local de ese equipo.
1
Respaldar la KNA
Copiar el archivo webManMov_XXXX.db desde la carpeta Documentos del equipo antes de cualquier modificación.
2
Cerrar Manager GO completamente y reabrir
Al reiniciar la sesión, el sistema sincroniza la KNA con el servidor. Si los datos del servidor están corregidos, esto puede resolver el problema sin edición manual.
3
Editar la KNA con DB Browser for SQLite (si persiste)
Abrir el archivo .db. Navegar a la tabla equivalente a RANGOS_ELECT. Corregir "último usado" o eliminar el duplicado. Guardar y cerrar antes de reconectar.
4
Reasociar el terminal y validar
Parámetros Locales › reasociar el terminal. Verificar que aparece "En Uso". Reingresar folios manuales si corresponde. Emitir documento de prueba y confirmar el número de folio.
Recomendaciones para futuras implementaciones
RecomendaciónDescripción
Un usuario Go por terminalEvitar el mismo correo en múltiples equipos simultáneamente. Previene colisiones en la asignación de folios.
Cargar folios antes de crear terminalesEl pool en ACEPTA debe estar completamente cargado antes de asignar rangos. Así el sistema toma del lote correcto.
Verificar antes de la prueba masivaEjecutar los scripts de verificación para confirmar rangos correctos y sin duplicados antes de la prueba simultánea.
Respaldar KNA antes de corregirSiempre copiar el archivo KNA antes de modificar la base de datos del servidor o la KNA local.
Salir y entrar de GO tras correccionesEl cliente debe cerrar GO completamente y volver a ingresar para sincronizar la KNA local con los datos corregidos.
