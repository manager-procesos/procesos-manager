Alcance y Descripción del Procedimiento

Este procedimiento describe el proceso de reimplementación de una base de datos (BD) productiva. Se aplica cuando un cliente que ya opera en el sistema requiere una reconfiguración completa de su BD: limpieza de datos históricos, carga de nueva información de maestros, inventario y contabilidad, dejando la BD lista para operar como si fuera una implementación nueva.

A diferencia de una réplica simple, la reimplementación incluye:

  • Levantamiento detallado de necesidades con el cliente
  • Creación de nueva BD a partir de tablas seleccionadas de la BD original
  • Configuración y verificación completa por parte de Proyectos Onboarding
  • Carga de inventario inicial y apertura contable
  • Apoyo DTE si el cliente utiliza Acepta
  • Acompañamiento en la puesta en marcha

1. Levantamiento Inicial

El proceso se inicia con el levantamiento realizado por Comercial o GEX junto al cliente, recopilando sus necesidades mediante el checklist comercial. Una vez acordados los alcances, se genera la OT, la cual es enviada al área de Proyectos Onboarding, la cual programa el respectivo levantamiento de proyecto en base al checklist comercial.

2. Selección de Tablas y Solicitud a IT

Proyectos Onboarding, en conjunto con el cliente, revisa las tablas de la BD de origen y acuerda qué información se copiará a la nueva BD. Las tablas obligatorias se incluyen siempre.

4. Sesiones Requeridas – Reimplementación

A continuación, se detallan las sesiones mínimas para una reimplementación de BD (solo configuración, sin capacitación de módulos). Si el cliente requiere capacitación de algún módulo específico, se deben agendar sesiones adicionales según el módulo a capacitar.

SesiónNombreDescripciónDuración
1 Levantamiento Reunión inicial con el cliente para relevar sus necesidades, configuraciones actuales y definir el alcance de la reimplementación. Se completa el checklist comercial. 2 hrs
2 Revisión de BD Proyectos Onboarding revisa la BD creada: tablas obligatorias, parámetros, formatos y accesos. Se verifica que la base esté correctamente configurada. 2 hrs
3 Presentación de Ciclo Se presenta al cliente el flujo completo del sistema (ciclo de operación). Se realizan pruebas funcionales internas para validar el correcto funcionamiento de la BD. 2 hrs
4 Explicación Importadores Se instruye al cliente sobre el uso de importadores para la carga masiva de datos: inventario, apertura contable y documentos en los distintos módulos. 2 hrs
5 Carga de Inventario El cliente o el consultor realiza la carga del inventario inicial mediante el importador correspondiente. Se verifica la correcta recepción de los datos. 2 hrs
6 Carga Contable Se realiza la apertura contable mediante el importador. Se validan los saldos iniciales y el plan de cuentas queda operativo. 2 hrs
7 Apoyo DTE (si aplica) Solo si el cliente utiliza Acepta. Se configuran certificados digitales, rutas DTE y parámetros web. Se prueba la emisión de un documento electrónico en el SII. 2 hrs
8 Apoyo Puesta en Marcha Acompañamiento al cliente en el inicio de operaciones en la nueva BD. Se resuelven dudas, se verifica el funcionamiento general y se cierra formalmente el proyecto. 2 hrs
⚠ Nota: La sesión 7 (Apoyo DTE) aplica únicamente si el cliente utiliza Acepta para facturación electrónica.
