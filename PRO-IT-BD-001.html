Alcance y Descripción del Procedimiento

Este procedimiento describe el proceso para crear una réplica de una base de datos (BD) existente, sin incluir movimientos (sin stock ni apertura contable). Se utiliza cuando un cliente necesita partir con una nueva BD, similar a la anterior, pero vacía (sin movimientos).

El proceso se inicia con un levantamiento realizado por Comercial o GEX junto al cliente, identificando qué tablas se traspasarán. Una vez acordado, se solicita a IT la creación de la nueva BD. Las tablas marcadas como OBLIGATORIAS se incluyen siempre; las demás, según acuerdo con el cliente.

⚠ Importante: Este procedimiento NO incluye carga de stock ni apertura contable.

1. Levantamiento y Selección de Tablas

Comercial o GEX, en conjunto con el cliente, revisa las tablas de la BD de origen y acuerda qué información se copiará a la nueva BD. Una vez definido, se solicita a IT la creación. Las tablas obligatorias se incluyen siempre, independientemente del tipo de réplica.

Nota: Si el cliente tiene formatos personalizados (facturas, guías, órdenes de compra), comparar con la BD original y habilitar los formatos requeridos antes de avanzar.

4. Sesiones Requeridas

Para una réplica de BD sin movimientos, se requiere mínimo 1 sesión de verificación y configuración.

SesiónNombreDescripciónDuración
1 Verificación y Configuración Revisión completa de la BD creada: tablas, parámetros, FTP, Ramaflex y prueba de ciclos. 2 hrs

Si el cliente requiere capacitación de algún módulo, se deben agendar sesiones adicionales según el módulo a capacitar.
