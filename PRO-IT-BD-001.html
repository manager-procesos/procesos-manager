1
Alcance y Descripción del Procedimiento
Este procedimiento describe el proceso para crear una réplica de una base de datos (BD) existente, sin incluir movimientos. Se utiliza cuando un cliente necesita partir con una nueva BD, similar a la anterior, pero vacía de movimientos.
El proceso se inicia con un levantamiento realizado por Comercial o GEX junto al cliente, identificando qué tablas se traspasarán. Una vez acordado, se solicita a IT la creación de la nueva BD. Las tablas marcadas como OBLIGATORIAS se incluyen siempre; las demás, según acuerdo con el cliente.
⚠
Importante: Este procedimiento NO incluye carga de stock ni apertura contable.
2
Levantamiento, Ambiente y Selección de Tablas
Comercial o GEX, en conjunto con el cliente, revisa las tablas de la BD de origen y acuerda qué información se copiará. Las tablas OBLIGATORIAS se incluyen siempre, independientemente del tipo de réplica.

BD
2.2 Tablas a Copiar
Las tablas en azul son OBLIGATORIAS y se copian siempre. Las demás se marcan SÍ o NO según lo acordado con el cliente.
N° Tabla Contenido OBLIGATORIA SÍ NO
OBLIGATORIA = se copia siempre SÍ = tabla incluida según acuerdo con cliente NO = no aplica para este cliente
3
Solicitud de Creación de BD a IT
Una vez definidas las tablas con el cliente, Comercial / GEX envía el siguiente correo a IT con el detalle completo de la solicitud.
Para:it@manager.cl
Asunto:Solicitud Creación BD – [NOMBRE CLIENTE] – [NOMBRE BD]
Área Solicitante:Comercial / GEX

Estimados, muy buenas tardes,

Junto con saludar, solicito lo siguiente:

1. Crear base denominada "[NOMBRE_BD]" y en el campo "Descripción" indicar: [descripción — máx. 30 caracteres]

2. Copiar desde base "[BD_ORIGEN]" a "[NOMBRE_BD]" las siguientes tablas (importante: debe ser desde esa base, no otra):

N° Tabla Contenido
AMBIENTE : \\MANAGER.LOCAL\MG\[código]\MANAGER8\COMUNES\
BD          : [código BD]

Nota: La base nueva debe quedar actualizada a la última versión disponible del sistema.

Estaré atenta a sus comentarios. Saludos cordiales.

4
Verificación Post-Creación – Proyectos Onboarding
Una vez que IT confirma la creación de la BD, el consultor de Onboarding recibe la tarea mediante la OT asignada y valida los siguientes puntos antes de presentar la BD al cliente.
Verificación SÍ NO Observaciones
⚠
Formatos de impresión: si el cliente tiene formatos personalizados (facturas, guías, órdenes de compra), comparar con la BD original y habilitar los formatos requeridos antes de avanzar.
5
Sesiones Requeridas
Para una réplica de BD sin movimientos se requiere mínimo 1 sesión de verificación y configuración. Si el cliente requiere capacitación de algún módulo, se deben agendar sesiones adicionales según el módulo a capacitar.
