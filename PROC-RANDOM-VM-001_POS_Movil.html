1. Objetivo y alcance

App para vendedores en terreno conectada vía API al ERP Manager Time. Permite crear Notas de Venta, consultar clientes y catálogo de productos. Opera online y offline.

AccesoGoogle Play Store → "Random Móvil"

3. Configuración previa en Manager ERP
7 pasos · Responsabilidad del consultor
1
Crear vendedor en tabla de personal
Personal → Ingresar. Código 3 caracteres (ej: VRB), Nombre, Apellido, e-mail. Grupo = Ventas. Marcar checkbox Vendedor. Asignar Sucursal y Bodega.
ERP
2
Crear usuario y asociar a grupo Ventas
Usuarios → Ingresar. Mismo código del vendedor. Clave Secreta. Grupo = Ventas. Verificar licencia disponible (Nominal o Concurrente) antes de crear. Si no hay licencias disponibles, escalar al equipo de desarrollo.
ERP
3
Crear listas de precios
Módulo Múltiple Lista de Precio. Crear: (a) Lista Offline código LOFF — precio referencial sin internet; (b) Lista del cliente con código propio. Si hay descuentos, se reflejan en la lista del cliente — no existe descuento por vendedor desde la app.
ERP
4
Asociar productos y verificar campo Unidad
Asociar cada producto a la lista con su precio. Crítico: verificar campo Unidad en ficha del producto (Productos y Servicios → campo Unidad). Sin este campo el producto no sincroniza. Solo aparecen en la app los productos con stock físico Y en lista activa.
ERPcampo crítico
5
Registrar clientes — asociar vendedor y lista de precios
Ficha del cliente → asignar campo Vendedor y campo Lista de Precios. Nota caso Ecovida: si hay múltiples listas por territorio, la llave de asignación es el RUT del cliente (un vendedor puede manejar varias LP).
ERP
6
Verificar condiciones de pago del cliente
Ficha del cliente → condiciones de pago: confirmar Glosa de Pago, Forma de Pago y Vendedor/Comisionista. Bug conocido: si sale "CONTADO EFECTIVO" en todas las NV independiente del cliente, coordinar con Rossi y Roberto León para revisar servicio API getDetalleSucursal.
ERPbug conocido
7
Solicitar acceso a plataforma Random
Solicitar a Rossi Colina (rcolina@random.cl) la creación del usuario y contraseña. Formato login: VRB@empresa. Al registrar la máquina en la app se ingresa solo VRB sin el @empresa.
Random

6. Imágenes de productos

Las imágenes se ven en el listado de productos de la app pero no se pueden descargar. No se extraen del ERP — el flujo pasa por Random.

Formato
JPG únicamente
Nombre del archivo
Exactamente el código KOPR del producto — ej: ABcd123.jpg
Peso máximo
200 KB por archivo
Entrega
Todos en una sola carpeta (Google Drive u otro medio compartido)
Quién carga
Equipo Random (César · Roblé Labs) — no el consultor ni el cliente
Coordinación
El consultor actúa de intermediario — comparte la carpeta con Random
Instrucción exacta para el cliente: "Nombre del archivo = código KOPR del producto. Formato JPG, máximo 200 KB. Dejar todos en una carpeta y compartirla con nosotros."
7. Geolocalización — aclarar ANTES de iniciar
✓ Disponible
La app registra la ubicación GPS de cada visita. Se visualiza al día siguiente en Power BI (Mapa de Visitas, Seguimiento de Vendedores). Es geolocalización diferida.
✗ No disponible
No existe seguimiento en tiempo real. El jefe de ventas NO puede ver la posición actual de los vendedores. Funcionalidad en línea no está desarrollada.
Lección aprendida — caso Grupo Latyn: El cliente fue informado comercialmente de que habría geolocalización en tiempo real (Marcelo Baeza). Esto generó conflicto en la implementación. Confirmar alcance por escrito antes de iniciar cualquier proyecto.
Compatibilidad de dispositivos — caso Grupo Latyn: Los equipos Honor funcionan solo en inglés y no son compatibles con la app. El cliente tuvo que cambiar todos los equipos a Samsung Android. Rossi confirmó que Samsung no presenta problemas. Verificar modelo del dispositivo antes de instalar.
